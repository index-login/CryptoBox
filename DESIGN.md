# CryptoBox 设计文档

## 项目概述

CryptoBox 是一个纯前端的加解密工具集合，部署在 GitHub Pages 上作为静态页面运行。所有计算均在浏览器端完成，不涉及任何服务端交互，确保数据安全。

主要面向网络安全从业者、CTF 选手、开发人员等，提供常用的编码/解码、加密/解密、哈希计算等工具。

## 技术选型

| 项目 | 选择 | 理由 |
|------|------|------|
| 框架 | 纯 HTML + CSS + JavaScript | 零依赖，GitHub Pages 直接托管，加载快 |
| UI 框架 | Tailwind CSS (CDN) | 快速构建美观界面，无需构建步骤 |
| 加密库 | CryptoJS / Web Crypto API | 浏览器原生支持 + 成熟社区库 |
| 国密库 | sm-crypto | SM2/SM3/SM4 国密算法实现 |
| 部署 | GitHub Pages | 免费、稳定、自动部署 |
| 构建 | 无需构建 | 纯静态文件，直接运行 |

## 痛点解决方案 & 智能处理

基于同类工具（CyberChef、在线加解密网站等）的常见不足，CryptoBox 需要重点解决以下痛点：

### 1. 智能输入预处理

用户粘贴进来的密文往往不是"干净"的，可能混杂各种编码和干扰字符。

| 处理项 | 说明 |
|--------|------|
| URL 编码自动还原 | 检测并自动去除 `%3D`→`=`、`%2B`→`+` 等 URL 编码 |
| 转义字符处理 | 自动识别 `\n`、`\t`、`\x41`、`\u0041`、`\\` 等转义序列 |
| 空白字符清理 | 可选去除多余空格、换行、制表符 |
| 前缀去除 | 自动去除 `0x`、`\x`、`0b` 等常见前缀 |
| 引号/括号剥离 | 去除包裹的引号 `"..."` 或方括号 `[...]` |
| 格式自动识别 | 智能判断输入是 Hex、Base64、纯文本还是其他格式 |

每个工具的输入框旁提供「智能清理」开关，用户可选择启用/关闭。

### 2. 编码自动识别 (Magic 模式)

提供独立的「Magic」工具页面：
- 输入未知编码文本，自动尝试所有可能的解码方式
- 按置信度排序展示结果
- 支持多层嵌套自动解码（如 Base64 → URL decode → Hex → 明文）
- 展示解码路径，方便用户理解编码层次

### 3. 流水线/链式处理 (Recipe)

- 支持将多个工具操作串联为一条流水线
- 拖拽排序操作步骤
- 上一步的输出自动作为下一步的输入
- 可保存常用 Recipe 到 localStorage
- 可通过 URL 参数分享 Recipe

典型场景：`URL Decode → Base64 Decode → AES Decrypt → UTF-8 输出`

### 4. 输入输出格式灵活性

所有工具的输入/输出统一支持以下格式切换：

| 格式 | 示例 |
|------|------|
| UTF-8 文本 | `Hello World` |
| Hex 字符串 | `48656c6c6f` |
| Base64 | `SGVsbG8=` |
| Hex (带空格) | `48 65 6c 6c 6f` |
| Hex (带0x前缀) | `0x48 0x65 0x6c` |
| 二进制字符串 | `01001000 01100101` |
| Raw bytes (文件上传) | 文件拖拽输入 |

### 5. 加密参数完整性

#### Padding 模式
对称加密工具必须支持完整的 Padding 选项：
- PKCS7 / PKCS5（默认）
- NoPadding
- ZeroPadding
- ISO10126
- ANSIX923

#### 密钥/IV 格式
- 明确提供格式选择下拉：UTF-8 文本 / Hex / Base64
- 密钥长度实时校验与提示（如 AES-128 需要 16 字节，当前输入 N 字节）
- IV 长度校验（如 AES CBC 模式 IV 必须 16 字节）
- 不足时提示，超出时警告

#### 输出格式
- 加密结果支持输出为：Base64 / Hex / Raw
- 解密结果支持显示为：UTF-8 / Hex / Base64

### 6. 批量处理模式

- 支持多行文本逐行处理（每行独立编码/解码/哈希）
- 批量模式开关，开启后输入框按行分割处理
- 结果逐行对应输出
- 适用场景：批量 Base64 解码、批量计算哈希、字典生成

### 7. 操作历史记录

- 自动保存最近 50 条操作记录到 localStorage
- 记录内容：工具名称、输入摘要、参数配置、输出结果、时间戳
- 支持一键恢复历史操作
- 支持清空历史

### 8. URL 分享功能

- 将当前操作状态（工具 + 输入 + 参数）编码到 URL hash 中
- 点击「分享」按钮生成链接
- 他人打开链接自动还原操作状态和结果
- 敏感数据（密钥等）可选是否包含在分享链接中

### 9. 错误处理与提示

- 解密失败时给出可能原因提示（密钥错误？Padding 不对？编码格式不匹配？）
- 输入格式不合法时高亮错误位置
- 对常见错误提供修复建议（如 Base64 长度不是4的倍数，提示补 `=`）

### 10. 字节序与编码细节

- 涉及数值转换的工具（IP转换、整数编码）支持大端/小端选择
- 字符编码选择：UTF-8 / GBK / Latin-1 / ASCII
- Hex 显示支持大写/小写切换

## UI 设计

### 布局

- **顶部导航栏**：项目名称 + GitHub 链接
- **左侧边栏**：工具分类目录（可折叠）
- **主内容区**：当前工具的操作界面
- **响应式设计**：移动端侧边栏变为顶部下拉菜单

### 交互规范

- 每个工具统一布局：输入区 → 操作按钮 → 输出区
- 支持一键复制输出结果
- 支持清空输入/输出
- 输入变化时实时计算（可选）
- 暗色主题为主，符合安全工具审美
- 工具间支持快速跳转

### 配色方案

- 主色调：深色背景 (#1a1a2e, #16213e)
- 强调色：青色 (#0f3460, #00d2d3)
- 文字色：浅灰 (#e0e0e0)
- 成功/错误提示：绿色/红色

## 安全考虑

1. **纯前端运行**：所有数据处理在浏览器本地完成，不发送任何数据到服务器
2. **无第三方追踪**：不引入 Google Analytics 等追踪脚本
3. **CSP 策略**：设置合理的 Content Security Policy
4. **离线可用**：考虑添加 Service Worker 支持离线使用

## 部署说明

1. 将代码推送到 GitHub 仓库
2. 在仓库 Settings → Pages 中选择部署分支（main）
3. 选择根目录 `/` 作为发布源
4. 访问 `https://<username>.github.io/CryptoBox/`

## 扩展性

- 模块化设计，每个工具独立一个函数/文件，方便后续添加新工具
- 统一的工具注册机制，新增工具只需：
  1. 编写工具逻辑函数
  2. 在配置中注册工具名称、分类、描述
  3. 自动生成对应 UI
